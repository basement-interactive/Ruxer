# Fluxer Desktop — Final Completion Plan

Living document. Updated after each slice/item. Status: `TODO` → `IN PROGRESS` → `DONE` (or `BLOCKED` / `SKIPPED`).

## Locked decisions

- **REST client:** Hybrid — regenerate types from the OpenAPI spec with progenitor, but keep the existing hand-written transport + rate limiter in `core/`. Generated types replace hand-written `models.rs` structs; `http.rs` + `ratelimit.rs` stay.
- **Token storage:** `keyring` crate (OS keychain).
- **Leaked `.env` token:** Remove from `.env` on disk, leave `.env.example`, flag with a comment. User handles rotation on fluxer.app.
- **Execution:** all six slices (A–F), in order.
